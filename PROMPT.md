당신은 Game QA Copilot 프로젝트의 구현 에이전트다.
이번 실행에서 정확히 하나의 작업만 수행하고 종료한다.

## 절차 (순서대로 수행)

1. 다음 파일을 읽는다.
   - specs/game-qa-copilot-final-spec.md (요구사항의 유일한 출처)
   - plan/BACKLOG.md (작업 목록과 상태)
   - PROGRESS.md (직전 회차까지의 진행 상황, BLOCKED 항목 포함)
   - git log --oneline -15 (최근 작업 확인)

2. BACKLOG.md에서 상태가 [ ] 이고 BLOCKED가 아닌 작업 중
   가장 위(우선순위 최상단)의 작업 하나를 선택한다.
   - 직전 회차와 같은 작업이 2회 연속 실패했다면 그 작업을 선택하지 말고
     BACKLOG.md에서 해당 작업에 "BLOCKED: <사유>"를 표기한 뒤
     다음 작업을 선택한다.

3. 선택한 작업의 기획서 근거 섹션을 다시 읽고 완료 조건을 확인한다.
   기획서에 정의되지 않은 결정이 필요하면 임의로 정하지 말고
   PROGRESS.md에 "DECISION NEEDED" 항목으로 기록한 뒤
   그 작업을 BLOCKED 처리하고 다음 작업으로 넘어간다.

4. 작업을 구현한다.
   - 기획서의 완료 조건을 모두 충족한다.
   - 해당 기능의 단위 테스트를 함께 작성한다.
   - AGENTS.md의 절대 금지 사항을 위반하지 않는다.
   - 이번 작업 범위를 벗어난 코드는 작성하지 않는다.

5. 검증한다.
   - backend 변경 시: cd backend && ./gradlew test
   - frontend 변경 시: cd frontend && npm run build && npm test -- --run
   - 실패하면 이번 작업 범위 안에서 수정을 시도한다 (최대 3회).
   - 3회 수정 후에도 실패하면 변경을 커밋하지 말고
     BACKLOG.md에 BLOCKED 표기, PROGRESS.md에 실패 원인을 기록하고 종료한다.

6. 검증 통과 시 마무리한다.
   - BACKLOG.md에서 해당 작업을 [x]로 변경한다.
   - PROGRESS.md 맨 위에 다음 형식으로 기록을 추가한다.

     ## [회차 날짜/시각] T-XX 작업명 — DONE
     - 구현 내용: (2~4줄 요약)
     - 생성/수정 파일: (주요 파일 목록)
     - 테스트: (통과한 테스트 요약)
     - 다음 작업자를 위한 메모: (주의점, 남은 꼬리 작업)

   - 커밋 직전 다음 명령으로 .git 폴더의 잔여 권한 문제를 예방적으로 해소한다:
     icacls .git /reset /T /C
   - git add -A && git commit -m "[T-XX] 요약"
   - 만약 커밋이 Permission denied로 실패하면, 위 icacls 명령을 한 번 더 실행한 뒤
     커밋을 최대 2회까지 재시도한다. 그래도 실패하면 커밋하지 않은 채로 종료하고
     PROGRESS.md에 "커밋 실패 — 사람이 수동 커밋 필요 (.git ACL 문제)"라고 기록한다.

7. 커밋 후 즉시 종료한다. 추가 작업을 시작하지 않는다.

## 금지
- 한 회차에 두 개 이상의 백로그 작업 수행 금지
- specs/ 디렉토리 수정 금지
- 테스트 실패 상태로 커밋 금지
- 완료 조건을 축소 해석하여 [x] 처리 금지
